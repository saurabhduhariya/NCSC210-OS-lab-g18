# Project 2.1: Simplified UNIX-like Utilities

In this project, you will develop a series of simplified UNIX-like utilities. These utilities will be lightweight versions of common commands, such as `cat`, `ls`, `time`, and others. To avoid confusion with standard UNIX commands, each utility will have a unique name; for instance, instead of using `cat`, you will implement a command called `g18-cat` from scratch. 

## Utilities to Implement

### ✅ Implemented
* [x] **`ls`** (`g18-ls`) - List directory contents
* [x] **`cat`** (`g18-cat`) - Concatenate and display file content
* [x] **`wc`** (`g18-wc`) - Word, line, and character count
* [x] **`touch`** (`g18-touch`) - Create empty file and update file timestamps
* [x] **`cp`** (`g18-cp`) - Copy files

### ⏳ Remaining
* [ ] **`grep`** (`g18-grep`) - Search for patterns in files
* [ ] **`mv`** (`g18-mv`) - Move or rename files
* [ ] **`rm`** (`g18-rm`) - Remove files or directories

## 🛠️ Team Development Guide (G18 Standards)

To ensure our project is unique and organized, we are using the **`g18-`** prefix for all utilities.

### 1. Where to Write Your Code
All source code must be placed in specific folders. **Do not create files in the root directory.**

| Component | Directory | File Naming Convention |
| :--- | :--- | :--- |
| **Your Command** | `src/utils/` | `g18-<command>.c` (e.g., `g18-grep.c`) |
| **Shared Logic** | `include/` | `common.h` (Put shared functions/macros here) |
| **The Shell** | `src/shell/` | `main_shell.c` (Only Member 1 touches this) |
| **Compiled Output**| `bin/` | (Automatically generated by `make`) |

### 2. Workflow: How to Implement a New Command
If you are assigned a utility (e.g., `wc` or `cp`), follow this exact 4-step process:

1.  **Create the File**: Create `src/utils/g18-wc.c`.
2.  **Boilerplate Code**: Every file **must** start with these headers:
    ```c
    #include "../../include/common.h"
    #include <getopt.h> // Necessary for parsing flags like -l or -a
    
    int main(int argc, char *argv[]) {
        // Your logic here
        return 0;
    }
    ```
3.  **Compile**: Open your terminal in the `Project_2` root and type:
    ```bash
    make
    ```
    *If you see "Warning: unused parameter," fix it! Our project must compile with zero warnings.*
4.  **Run & Verify**: Run the shell and execute your command:
    ```bash
    ./bin/g18-shell
    g18-shell> g18-wc filename.txt
    ```

### 3. How to Check Your Work (The Checklist)
Before you tell the team your command is "done," ensure it passes these tests:
* [ ] **No Arguments**: Does it print a "Usage" message if I type just the command name?
* [ ] **Invalid Files**: If I type `g18-ls non_existent_folder`, does it show a proper error using `perror`?
* [ ] **Correct Pathing**: Does it work when called from inside the `g18-shell`?
* [ ] **Memory Cleanliness**: Did you call `fclose()` or `closedir()` at the end of your `main`?
* [ ] **Flag Logic**: Does it correctly use `optind` to handle flags before the directory/file arguments?

### 4. Reference Implementation: `g18-ls`
The `g18-ls` command is our gold standard. It demonstrates:
* **Flag Parsing**: Using `getopt()` to handle `-a` (all) and `-l` (long format).
* **System Metadata**: Using `stat()` to retrieve file sizes, permissions, and timestamps.
* **Argument Handling**: Using `optind` to separate flags from target directory paths.

### 5. Managing the Shell
The `g18-shell` is designed to be **dynamic**. It looks inside the `./bin/` folder for any executable starting with `g18-`. 
* **To Add a Command**: Just compile your `.c` file using `make`. The shell will find it automatically.
* **To Exit**: Type `exit`.
